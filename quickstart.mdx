## Welcome to Food For All DC

This guide will walk you through the essential steps to start using the platform, from logging in to scheduling your first delivery.

<Note>
  This quick start is for **platform users**. If you're a developer setting up the application locally, see the [Installation Guide](/installation).
</Note>

## Step 1: Access the Platform

<Steps>
  <Step title="Navigate to the Login Page">
    Open your web browser and navigate to your Food For All DC instance URL.
  </Step>
  
  <Step title="Sign In">
    Enter your credentials provided by your administrator. The platform uses Firebase Authentication for secure access.
    
    <Note>
      If you've forgotten your password, click the "Forgot Password" link on the login page to reset it.
    </Note>
  </Step>
  
  <Step title="Verify Your Role">
    After logging in, you'll see features based on your role:
    - **Admin**: Full access to all features
    - **Manager**: Calendar, routes, reports, client management
    - **ClientIntake**: Client profiles and basic calendar
    - **Driver**: Assigned delivery routes
  </Step>
</Steps>

## Step 2: Add Your First Client

<Steps>
  <Step title="Navigate to Client Management">
    From the main navigation, access the client management section or click "Add Client" from the calendar.
  </Step>
  
  <Step title="Fill Out the Intake Form">
    Complete the required fields:
    
    **Basic Information**
    - First Name and Last Name (required)
    - Address, City, State, ZIP Code (required)
    - Phone number (required, at least 10 digits)
    - Email (optional)
    
    **Household Details**
    - Number of adults, seniors, and children
    - Head of household type
    - Gender, ethnicity, and language
    
    **Delivery Information**
    - Delivery instructions
    - Dietary restrictions (checkboxes for common restrictions)
    - Food allergens (comma-separated list)
    
    <Warning>
      The address will be automatically geocoded to determine the DC ward. Make sure to enter a complete, valid address.
    </Warning>
  </Step>
  
  <Step title="Set Delivery Schedule">
    Configure the client's delivery schedule:
    - **Start Date**: When deliveries should begin
    - **End Date**: When deliveries should end
    - **Recurrence**: None, Weekly, Bi-Weekly, Monthly, or Custom
    
    <Note>
      You can modify the schedule later by editing the client profile or adding individual deliveries.
    </Note>
  </Step>
  
  <Step title="Add Referral Information">
    Select or create a case worker/referral entity:
    - Case worker name
    - Organization
    - Contact information
    
    This helps track where clients are coming from.
  </Step>
  
  <Step title="Save the Profile">
    Click the save button. The system will:
    - Validate all required fields
    - Check for duplicate clients (same name and address)
    - Geocode the address to get coordinates and ward
    - Create the client profile in the database
    
    <Note>
      If you see a duplicate warning, review the existing client record before proceeding.
    </Note>
  </Step>
</Steps>

## Step 3: Schedule a Delivery

<Steps>
  <Step title="Open the Calendar">
    Navigate to the Calendar page. You'll see:
    - **Day View**: Detailed view of deliveries for a specific day
    - **Month View**: Overview of deliveries across the month
    
    Use the navigation buttons to change dates:
    - Previous/Next arrows to move by day or month
    - "Today" button to jump to the current date
  </Step>
  
  <Step title="Click Add Delivery">
    Click the "Add Delivery" button in the calendar header.
  </Step>
  
  <Step title="Select the Client">
    In the Add Delivery dialog:
    - Use the autocomplete field to search for the client
    - Type the client's name to filter the list
    - Select the client from the dropdown
    
    The system will load all client profiles for quick access.
  </Step>
  
  <Step title="Choose Delivery Dates">
    Select the delivery schedule:
    
    **One-Time Delivery**
    - Choose "None" for recurrence
    - Pick a single delivery date
    
    **Recurring Delivery**
    - Choose Weekly, Bi-Weekly, or Monthly
    - Set the start date
    - Set the end date (when to stop recurring)
    
    **Custom Dates**
    - Choose "Custom" recurrence
    - Select multiple specific dates from the calendar
    
    <Note>
      The system will check daily capacity limits and warn you if adding deliveries would exceed the configured limit.
    </Note>
  </Step>
  
  <Step title="Save the Delivery">
    Click "Add Delivery". The system will:
    - Filter out dates that already have a delivery for this client
    - Create delivery events in the database
    - Link all events in a recurring series with a `recurrenceId`
    - Update the calendar view
    
    You'll see a success message showing how many deliveries were added.
  </Step>
</Steps>

## Step 4: Organize Deliveries into Routes

<Steps>
  <Step title="Navigate to Delivery Routes">
    Go to the Delivery/Routes page from the main navigation.
  </Step>
  
  <Step title="Select a Delivery Date">
    Use the date picker to choose the date you want to plan routes for.
    
    You'll see a spreadsheet view with all scheduled deliveries for that date, including:
    - Client name, address, and phone
    - Dietary restrictions (color-coded)
    - Current cluster and driver assignments
  </Step>
  
  <Step title="Generate Clusters">
    Click the "Generate Clusters" button to automatically group deliveries by location:
    
    - Choose the number of clusters (routes)
    - The system uses a K-means clustering algorithm
    - Deliveries are grouped based on geographic coordinates
    
    <Note>
      The clustering algorithm calls a Python Cloud Function that uses the scikit-learn library for optimization.
    </Note>
    
    Each cluster is assigned a color on the map and in the spreadsheet.
  </Step>
  
  <Step title="Assign Drivers">
    For each cluster:
    - Select deliveries in the cluster (use checkboxes)
    - Click "Assign Driver"
    - Choose a driver from the dropdown
    - Click "Assign"
    
    Or manually assign drivers in the spreadsheet by selecting from the driver dropdown for each row.
  </Step>
  
  <Step title="Assign Time Slots">
    Set delivery time windows:
    - Select deliveries
    - Click "Assign Time"
    - Choose a time slot (e.g., "10 AM - 12 PM", "12 PM - 2 PM")
    - Click "Assign"
  </Step>
  
  <Step title="View on Map">
    Click "View Map" to see:
    - Color-coded markers for each cluster
    - Client addresses with popup information
    - Visual confirmation that clusters make geographic sense
    
    You can manually adjust cluster assignments if needed.
  </Step>
  
  <Step title="Export Routes">
    Generate route sheets for drivers:
    
    - Click "Export" and choose your format:
      - **Standard CSV**: All delivery details
      - **DoorDash CSV**: Compatible with DoorDash Drive API
    
    The export includes:
    - Client name and address
    - Phone number
    - Delivery instructions
    - Dietary restrictions
    - Assigned driver and time slot
  </Step>
</Steps>

## Step 5: Generate Reports

<Steps>
  <Step title="Access Reports">
    Navigate to the Reports page from the main navigation.
  </Step>
  
  <Step title="Choose Report Type">
    Select from available reports:
    
    **Summary Report**
    - Total deliveries by date range
    - Unique clients served
    - Breakdown by demographics
    
    **Client Report**
    - Individual client delivery history
    - Frequency and patterns
    
    **Referral Agencies Report**
    - Clients by referral source
    - Organization-level metrics
    
    **Caseworker Report**
    - Deliveries by case worker
    - Individual case worker performance
  </Step>
  
  <Step title="Set Date Range">
    Use the date range picker to select:
    - Start date
    - End date
    
    The report will include all deliveries within this range.
  </Step>
  
  <Step title="Generate and Export">
    Click "Generate Report" to view the results in a table.
    
    Export options:
    - **CSV**: For Excel or Google Sheets
    - **PDF**: For printing or archiving (if configured)
    
    <Note>
      Reports are generated in real-time from the Firestore database, so they always reflect the latest data.
    </Note>
  </Step>
</Steps>

## Tips for Success

<Tip>
  **Keep client information up to date**: Regularly review and update client profiles, especially addresses and dietary restrictions. The geocoding system will automatically update ward assignments when addresses change.
</Tip>

<Tip>
  **Use tags effectively**: Create tags like "Senior", "Mobility Issues", "Prefers AM delivery" to quickly filter and identify special requirements.
</Tip>

<Tip>
  **Review capacity warnings**: The calendar shows visual warnings when daily delivery limits are exceeded. Adjust limits in the calendar settings or redistribute deliveries across days.
</Tip>

<Warning>
  **Avoid duplicate clients**: Always search for existing clients before creating a new profile. The system will warn you about duplicates, but it's best to check first.
</Warning>
