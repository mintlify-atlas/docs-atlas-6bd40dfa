## Overview

The `deleteUserAccount` callable function allows administrators and managers to delete user accounts from both Firebase Authentication and Firestore. The function enforces role-based access control to ensure only authorized users can perform deletions.

## Endpoint

```
https://us-central1-food-for-all-dc-caf23.cloudfunctions.net/deleteUserAccount
```

**Type:** Firebase Callable Function  
**Region:** `us-central1`

## Authentication

This is a Firebase Callable Function that automatically handles authentication. The caller must be authenticated with Firebase Auth.

## Authorization

**Required roles:** Admin or Manager

**Permission rules:**
- Admins can delete any non-admin user
- Managers can delete non-admin users (but NOT other admins)
- No user can delete their own account via this function
- Unauthenticated requests are rejected

## Request

<ParamField body="uid" type="string" required>
  The Firebase Auth UID of the user to delete.
  
  **Constraints:**
  - Must be a valid string
  - Cannot be the caller's own UID (self-deletion not allowed)
  - User must exist in Firebase Auth
</ParamField>

## Error Codes

| Code | Description |
|------|-------------|
| `unauthenticated` | No authentication context found (user not logged in) |
| `invalid-argument` | Missing or invalid `uid` parameter |
| `permission-denied` | Insufficient permissions (not admin/manager, self-deletion, or manager trying to delete admin) |
| `not-found` | User with specified UID does not exist in Firebase Auth |
| `internal` | Unexpected server error during deletion |

## Implementation Details

**Source:** `functions-python/main.py:73`

The function:
1. Validates the caller is authenticated
2. Validates the `uid` parameter is provided and is a string
3. Prevents self-deletion (caller cannot delete their own account)
4. Determines the caller's role from custom claims or Firestore `/users/{uid}` document
5. Checks the caller has admin or manager role
6. Determines the target user's role
7. Prevents managers from deleting admin accounts
8. Deletes the user from Firebase Authentication
9. Deletes the user's Firestore document from `/users/{uid}` collection
10. Returns success message

### Role Detection

The function checks for roles in the following order:

**Custom claims** (checked first):
- `role`
- `userRole`
- `user_type`
- `type`

**Firestore `/users/{uid}` document** (checked if no claim found):
- `role`
- `type`
- `userType`

Roles are normalized to lowercase and spaces replace underscores.

## Deletion Process

1. **Firebase Auth deletion** - User removed from Authentication
2. **Firestore deletion** - User document removed from `/users/{uid}`

**Note:** If Firestore deletion fails after Auth deletion succeeds, the function still returns success but logs a warning. The Auth account is considered the authoritative source.

## Best Practices

<Warning>
  User deletion is **irreversible**. Always confirm with the user before calling this function.
</Warning>

- Implement confirmation dialogs before deletion
- Log deletion events for audit trails
- Consider soft-deletes (marking users as inactive) instead of hard deletes
- Refresh user lists after successful deletion
- Handle errors gracefully and display user-friendly messages
