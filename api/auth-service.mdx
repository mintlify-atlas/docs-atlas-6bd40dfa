## Overview

The `AuthUserService` is a singleton service that handles all authenticated user-related operations with Firebase Authentication and Firestore. It manages user creation, deletion, role mapping, and provides real-time user data subscriptions.

## Getting Started

```typescript
import { authUserService } from '@/services/AuthUserService';

// The service is already instantiated as a singleton
const users = await authUserService.getAllUsers();
```

### subscribeToAllUsers

Subscribe to real-time updates for all users.

```typescript
public subscribeToAllUsers(
  onData: (users: AuthUserRow[]) => void,
  onError?: (error: ServiceError) => void
): () => void
```

<ParamField path="onData" type="(users: AuthUserRow[]) => void" required>
  Callback function invoked when user data changes
</ParamField>

<ParamField path="onError" type="(error: ServiceError) => void">
  Optional error handler for subscription errors
</ParamField>

<ResponseField name="return" type="() => void">
  Unsubscribe function to stop listening to updates
</ResponseField>

**Example:**
```typescript
const unsubscribe = authUserService.subscribeToAllUsers(
  (users) => {
    console.log(`Received ${users.length} users`);
    // Update UI with latest user data
  },
  (error) => {
    console.error('Subscription error:', error);
  }
);

// Later, when component unmounts
unsubscribe();
```

<Warning>
Always call the unsubscribe function when your component unmounts to prevent memory leaks.
</Warning>

**React Example:**
```typescript
import { useEffect, useState } from 'react';
import { authUserService } from '@/services/AuthUserService';
import { AuthUserRow } from '@/types';

function UserManagement() {
  const [users, setUsers] = useState<AuthUserRow[]>([]);

  useEffect(() => {
    const unsubscribe = authUserService.subscribeToAllUsers(
      (updatedUsers) => setUsers(updatedUsers),
      (error) => console.error('User subscription error:', error)
    );

    return () => unsubscribe();
  }, []);

  return (
    <div>
      {users.map(user => (
        <div key={user.uid}>
          {user.name} - {user.role}
        </div>
      ))}
    </div>
  );
}
```

<ParamField path="uid" type="string" required>
  The unique identifier of the user
</ParamField>

<ParamField path="onData" type="(user: AuthUserRow | null) => void" required>
  Callback function invoked when user data changes
</ParamField>

<ParamField path="onError" type="(error: ServiceError) => void">
  Optional error handler for subscription errors
</ParamField>

<ResponseField name="return" type="() => void">
  Unsubscribe function to stop listening to updates
</ResponseField>

**Example:**
```typescript
const unsubscribe = authUserService.subscribeToUserById(
  'user-uid-123',

      console.log(`User updated: ${user.name}`);

      console.log('User not found or deleted');
    }

  (error) => {
    console.error('Subscription error:', error);
```

### createUser

Create a new authenticated user with Firebase Authentication and Firestore.

```typescript
public async createUser(
  userData: Omit<AuthUserRow, "id" | "uid">,
  password: string
): Promise<string>
```

<ParamField path="userData" type="Omit<AuthUserRow, 'id' | 'uid'>" required>
  User data without the ID/UID fields (Firebase generates these)
</ParamField>

<ParamField path="password" type="string" required>
  Password for the new user account
</ParamField>

<ResponseField name="return" type="Promise<string>">
  The UID of the newly created user
</ResponseField>

**Example:**
```typescript
const newUser = {
  name: 'Jane Doe',
  email: 'jane.doe@example.com',
  phone: '202-555-0199',
  role: UserType.Manager
};

const userId = await authUserService.createUser(newUser, 'SecurePassword123!');
console.log(`Created user with UID: ${userId}`);
```

**Error Handling:**
```typescript
import { ServiceError } from '@/utils/serviceError';

try {
  const userId = await authUserService.createUser(newUser, password);
  console.log('User created successfully:', userId);
} catch (error) {
  if (error instanceof ServiceError) {
    if (error.message.includes('email-already-in-use')) {
      console.error('This email is already registered');
    } else if (error.message.includes('weak-password')) {
      console.error('Password is too weak');
    } else {
      console.error('Error creating user:', error.message);
    }
  }
}
```

<Note>
The current user's authentication context is automatically restored after creating a new user to prevent accidental sign-out.
</Note>

**User Role Mapping:**
Roles are mapped from `UserType` enum to display strings:
- `UserType.Admin` → "Admin"
- `UserType.Manager` → "Manager"
- `UserType.ClientIntake` → "Client Intake"

### deleteUser

Delete a user account from both Firebase Authentication and Firestore.

```typescript
public async deleteUser(uid: string): Promise<void>
```

<ParamField path="uid" type="string" required>
  The unique identifier of the user to delete
</ParamField>

**Example:**
```typescript
await authUserService.deleteUser('user-uid-123');
console.log('User deleted successfully');
```

**With Confirmation:**
```typescript
const confirmDelete = confirm(
  `Are you sure you want to delete this user? This action cannot be undone.`
);

if (confirmDelete) {
  try {
    await authUserService.deleteUser(userId);
    console.log('User deleted');
  } catch (error) {
    if (error instanceof ServiceError) {
      console.error('Delete failed:', error.message);
    }
  }
}
```

<Warning>
This operation cannot be undone. The user will be removed from both Firebase Authentication and the Firestore users collection.
</Warning>

**Error Codes:**
- `permission-denied`: You don't have permission to delete this user
- `not-found`: User not found (may already be deleted)
- `invalid-argument`: Invalid request parameters

**Error Handling:**
```typescript
try {
  await authUserService.deleteUser(uid);
} catch (error) {
  if (error instanceof ServiceError) {
    switch (error.code) {
      case 'permission-denied':
        console.error('Insufficient permissions');
        break;
      case 'not-found':
        console.error('User already deleted');
        break;
      default:
        console.error('Delete failed:', error.message);
    }
  }
}
```

<Note>
This method calls the Firebase Cloud Function `deleteUserAccount` to handle the deletion securely.
</Note>

## Role Mapping

The service automatically maps between string role values and `UserType` enum:

```typescript
const mapRoleToUserType = (roleString: string): UserType => {
  switch (roleString?.toLowerCase()) {
    case 'admin':
      return UserType.Admin;
    case 'manager':
      return UserType.Manager;
    case 'client intake':
      return UserType.ClientIntake;
    default:
      return UserType.ClientIntake;
  }
};
```

## Best Practices

1. **Use Real-time Subscriptions**: For admin panels and user management UIs, use `subscribeToAllUsers` to get live updates when users are added, modified, or deleted.

2. **Password Requirements**: Ensure passwords meet Firebase's minimum requirements (at least 6 characters) to avoid "weak-password" errors.

3. **Error Handling**: Always handle potential errors when creating or deleting users, especially "email-already-in-use" errors.

4. **Cleanup Subscriptions**: Always call the unsubscribe function when components unmount.

5. **Role Validation**: Validate user roles on both client and server side to ensure proper access control.
