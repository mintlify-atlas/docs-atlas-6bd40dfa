## Overview

The `DeliveryService` is a singleton service that handles all delivery-related operations with Firebase Firestore. It manages delivery events, scheduling, limits, clustering, and provides optimized methods for calendar operations.

## Getting Started

```typescript
import DeliveryService from '@/services/delivery-service';

const deliveryService = DeliveryService.getInstance();
const events = await deliveryService.getAllEvents();
```

### getRecurringDeliveryDateRange

Get the earliest and latest delivery dates for a client's recurring deliveries.

```typescript
public async getRecurringDeliveryDateRange(
  clientId: string,
  recurrenceType: string
): Promise<{ earliest: Date | null; latest: Date | null }>
```

<ParamField path="clientId" type="string" required>
  The client ID to check
</ParamField>

<ParamField path="recurrenceType" type="string" required>
  The recurrence type (e.g., 'Weekly', 'BiWeekly', 'Monthly', 'None')
</ParamField>

<ResponseField name="earliest" type="Date | null">
  Earliest delivery date, or null if no deliveries found
</ResponseField>

<ResponseField name="latest" type="Date | null">
  Latest delivery date, or null if no deliveries found
</ResponseField>

**Example:**
```typescript
const { earliest, latest } = await deliveryService.getRecurringDeliveryDateRange(
  'client-123',
  'Weekly'
);

if (earliest && latest) {
  console.log(`Deliveries from ${earliest} to ${latest}`);
}
```

<Note>
Results are cached for 5 minutes to improve performance for calendar operations.
</Note>

</ResponseField>

## Event Emitter

The DeliveryService emits delivery change events that can be subscribed to:

```typescript
import { deliveryEventEmitter } from '@/utils/deliveryEventEmitter';

deliveryEventEmitter.subscribe((event) => {
  console.log('Delivery change:', event.reason);
  console.log('Impacted dates:', event.impactedDateKeys);
});
```

**Event Reasons:**
- `schedule-created`: Single event created
- `schedule-created-batch`: Multiple events created
- `schedule-updated`: Event updated
- `schedule-deleted`: Event deleted
- `schedule-batch-deleted`: Multiple events deleted

## Error Handling

All methods throw `ServiceError` exceptions:

```typescript
import { ServiceError } from '@/utils/serviceError';

try {
  await deliveryService.createEvent(eventData);
} catch (error) {
  if (error instanceof ServiceError) {
    console.error(`Error: ${error.message}`);
  }
}
```
