## Overview

The `cluster_deliveries_k_means` function groups delivery coordinates into clusters using constrained K-means clustering. This enables efficient route assignment by ensuring each driver receives a balanced number of deliveries.

## Endpoint

```
POST https://us-central1-food-for-all-dc-caf23.cloudfunctions.net/cluster_deliveries_k_means
```

**Region:** `us-central1`  
**Memory:** 512 MB  
**Timeout:** 300 seconds

## Authentication

This endpoint requires Firebase Authentication. Include a valid Firebase ID token in the Authorization header.

```
Authorization: Bearer <firebase-id-token>
```

## Request

<ParamField body="coords" type="array" required>
  Array of coordinate tuples `[latitude, longitude]` representing delivery locations.
  
  **Constraints:**
  - Maximum 5000 coordinates per request
  - Each coordinate must be a tuple of two floats: `[lat, lng]`
</ParamField>

<ParamField body="drivers_count" type="integer" required>
  Number of drivers (clusters) to create.
  
  **Constraints:**
  - Must be greater than 0
  - Maximum 500 drivers
  - If greater than the number of deliveries, automatically adjusted to match delivery count
</ParamField>

<ParamField body="min_deliveries" type="integer" required>
  Minimum number of deliveries per driver.
  
  **Constraints:**
  - Must be greater than 0
  - Cannot exceed `max_deliveries`
</ParamField>

<ParamField body="max_deliveries" type="integer" required>
  Maximum number of deliveries per driver.
  
  **Constraints:**
  - Must be greater than 0
  - Must be greater than or equal to `min_deliveries`
</ParamField>

```python Python
import requests

def cluster_deliveries(coords, drivers_count, min_deliveries, max_deliveries, id_token):
    url = 'https://us-central1-food-for-all-dc-caf23.cloudfunctions.net/cluster_deliveries_k_means'
    headers = {
        'Content-Type': 'application/json',
        'Authorization': f'Bearer {id_token}'
    }
    payload = {
        'coords': coords,
        'drivers_count': drivers_count,
        'min_deliveries': min_deliveries,
        'max_deliveries': max_deliveries
    }
    
    response = requests.post(url, json=payload, headers=headers)
    return response.json()

# Usage

    ],

    max_deliveries=2,
    id_token=firebase_id_token
)
print(result['clusters'])
```

## Response

<ResponseField name="clusters" type="object">
  Object mapping cluster IDs (driver numbers) to arrays of delivery indices.
  
  - Keys are cluster IDs as strings (e.g., "1", "2", "3")
  - Values are arrays of indices from the input `coords` array
  - Each index represents a delivery assigned to that driver
</ResponseField>

## Implementation Details

**Source:** `functions-python/clustering.py:236`

The function:
1. Validates authentication token
2. Validates request payload and parameter constraints
3. Converts lat/lng coordinates to 3D Cartesian coordinates for accurate distance calculation
4. Applies constrained K-means clustering using the `k-means-constrained` library
5. Groups delivery indices by cluster ID
6. Returns cluster assignments as a dictionary

### Algorithm

The function uses **constrained K-means clustering** with:
- **Cartesian conversion:** Lat/lng pairs are converted to 3D Cartesian coordinates using Earth's radius (6371 km) for accurate geographic distance calculations
- **Size constraints:** Each cluster respects `min_deliveries` and `max_deliveries` constraints
- **Automatic adjustment:** If `drivers_count` exceeds the number of deliveries, it's automatically reduced
- **Random state:** Fixed at 42 for reproducible results

## Rate Limits

- **Maximum coordinates:** 5000 per request
- **Maximum drivers:** 500 per request
- **Function timeout:** 300 seconds

## Notes

- Cluster IDs start at "1" (not "0")
- If the number of drivers exceeds deliveries, the function automatically adjusts to create one cluster per delivery
- The clustering algorithm ensures balanced workload distribution while minimizing total distance
- Coordinates are converted to Cartesian space to account for Earth's curvature in distance calculations
