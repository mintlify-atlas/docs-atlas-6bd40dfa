## Directory Details

### `/auth` - Authentication

**Purpose:** Authentication logic and route protection

```
auth/
├── AuthProvider.tsx       # Auth context provider
├── ProtectedRoute.tsx     # Role-based route protection
└── firebaseConfig.ts      # Firebase initialization
```

**Key Files:**

**AuthProvider.tsx** - Manages user authentication state and role

```typescript
import { useAuth } from '../auth/AuthProvider';

const { user, userRole, loading } = useAuth();
```

**ProtectedRoute.tsx** - Wraps routes requiring authentication

```typescript
// Example usage in routesConfig.tsx
<Route element={<ProtectedRoute allowedRoles={[UserType.Admin]} />}>
  <Route path="/admin" element={<AdminPage />} />
</Route>
```

**Service Pattern:**

All services use the singleton pattern:

```typescript
// client-service.ts
class ClientService {
  private static instance: ClientService;

  private constructor() {}

  static getInstance(): ClientService {
    if (!ClientService.instance) {
      ClientService.instance = new ClientService();
    }
    return ClientService.instance;
  }

  async getClientById(uid: string): Promise<ClientProfile> {
    try {
      const docRef = doc(db, "client-profile2", uid);
      const docSnap = await getDoc(docRef);
      
      if (!docSnap.exists()) {
        throw new Error("Client not found");
      }
      
      return docSnap.data() as ClientProfile;
    } catch (error) {
      throw formatServiceError(error, "Failed to fetch client");
    }
  }

  // More methods...
}

export default ClientService;
```

**Usage:**

```typescript
import ClientService from '../services/client-service';

const clientService = ClientService.getInstance();
const client = await clientService.getClientById(uid);
```

<Note>
Always use `formatServiceError()` from `utils/serviceError.ts` when throwing errors in services.
</Note>

### `/hooks` - Custom Hooks

**Purpose:** Reusable React hooks

```

```

**Example Hook:**

```typescript
// usePerformance.ts
import { useEffect } from 'react';
import { performanceMonitor } from '../services/performance-monitor';

export const usePerformance = (componentName: string) => {
  useEffect(() => {
    const startTime = performance.now();
    
    return () => {
      const duration = performance.now() - startTime;
      performanceMonitor.logRender(componentName, duration);
    };
  }, [componentName]);
};
```

**apiKeys.ts** - Centralized API key management:

```typescript
function getRequiredEnv(key: string): string {
  const value = process.env[key];
  if (!value) {
    throw new Error(`Required environment variable ${key} is not set`);
  }
  return value;
}

export const firebaseConfig = {
  apiKey: getRequiredEnv("REACT_APP_FIREBASE_API_KEY"),
  authDomain: getRequiredEnv("REACT_APP_FIREBASE_AUTH_DOMAIN"),
  // ...
};
```

<Warning>

## Naming Conventions

### Files

- **Components:** `PascalCase.tsx` (e.g., `ClientProfile.tsx`)
- **Utilities:** `camelCase.ts` (e.g., `dateValidation.ts`)
- **CSS Modules:** `PascalCase.module.css` (e.g., `ClientProfile.module.css`)
- **Types:** `kebab-case-types.ts` (e.g., `client-types.ts`)

### Code

- **Components/Types/Enums:** `PascalCase`
- **Variables/Functions:** `camelCase`
- **Constants:** `UPPER_SNAKE_CASE`
- **Booleans:** `is/has/should` prefix (e.g., `isLoading`, `hasError`)
- **Event Handlers:** `handle/on` prefix (e.g., `handleClick`, `onSubmit`)
- **Hooks:** `use` prefix (e.g., `useAuth`, `useClientData`)
- **Services:** `Service` suffix (e.g., `ClientService`)

## Adding New Code

### Adding a Page

1. Create component: `src/pages/NewPage/NewPage.tsx`
2. Create styles: `src/pages/NewPage/NewPage.module.css`
3. Add route in `src/routesConfig.tsx`
4. Add to navigation in `src/pages/Base/Base.tsx`

### Adding a Service

1. Create file: `src/services/new-service.ts`
2. Use singleton pattern
3. Export in `src/services/index.ts`
4. Add error handling with `formatServiceError()`

### Adding Types

1. Create file: `src/types/feature-types.ts`
2. Define interfaces/types
3. Export in `src/types/index.ts`

### Adding Utilities

1. Create file: `src/utils/newUtility.ts`
2. Add utility functions
3. Export in `src/utils/index.ts`
